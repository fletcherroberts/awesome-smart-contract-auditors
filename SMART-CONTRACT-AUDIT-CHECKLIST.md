# Smart Contract Audit Checklist

Use this checklist if you are selecting a **smart contract audit company**, comparing **smart contract auditors**, or preparing to start an audit.

## Before choosing a firm

- define scope clearly
- document protocol architecture
- identify critical contracts
- outline upgrade paths and admin controls
- prepare tests and deployment assumptions

## When evaluating audit firms

- ask who will actually review the code
- ask how much review is manual
- ask what remediation support is included
- ask how retesting works
- ask for realistic timing
- ask how findings are prioritized
- ask for relevant prior experience

## Before the audit starts

- freeze scope where possible
- organize repositories cleanly
- prepare docs for protocol logic
- surface known risk areas early
- identify communication owners on both sides

## After findings are delivered

- triage issues by severity and exploitability
- confirm fixes with the auditors
- request retesting where needed
- document what remains out of scope
- plan launch only after remediation is understood
